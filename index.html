End-of-Life (EOL) Doctrine

When a piece of software is useful, there should never be an EOL doctrine. The intention for original creations is to achieve immoratlity ;).

At some point of time in the future, those projects may appear to be dead and abandon. The opposite will be true!

When this project reaches that stage, this project has matured to a level where maintenance is minimal or non-existent.
